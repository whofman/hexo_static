Octopress Hernieuwd

  • Gepost op: 2013-05-09
Na een lange onderbreking heb ik weer eens naar Octopress gekeken. Op de website was te zien dat de ontwikkelaars niets gedaan hadden. Om het definitieve systeem ook via mijn eigen domein op internet te kunnen benaderen, moest ik gebruik maken van een virtual host op een ander port dan de standaard 80. Mijn Blog Keus

  • Gepost op: 2012-10-18
Na met diverse systemen geëxperimenteerd te hebben en rekening te houden met mijn post Aandacht, heb ik gekozen voor database overeenkomstig de Pritlog sqlite3 database. Deze zal gevoed worden door Markdown editors via een Hazel script. De editors saven de file in ~/Dropbox/Notes met als filename bp_\.md. Een Hazel regel verplaatst iedere file met voornoemde filename naar de inbox van het database systeem. Deze wordt op zijn beurt gescanned door een cronjob post_it.php die de tekst file converteerd en toevoegd aan de database. Een tekst file dient te worden voorzien van een header met minimaal de Titel: optie: Titel:\ Datum:yyyy-mm-dd [hh:mm:ss] Type:post|page Tag:\ gescheiden door een spatie Publ:1|0 (ja/nee publiseren) Via een webpagina kan de database bekeken worden.
