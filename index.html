Kirby v2 Panel

  • Gepost op: 2014-09-26
Kirby v2 panel gaf write access fouten, in tegenstelling tot de v1 versie. De oplossing was het toewijzen van de ownership of een aantal directories aan Apache. sudo chown -R _www:staff content sudo chown -R _www:staff thumbs sudo chown -R _www:staff site/accounts sudo chown -R _www:staff site/cache sudo chown -R _www:staff assets/avatars en deze writable te maken. chmod -R 755 content chmod -R 755 thumbs chmod -R 755 site/accounts chmod -R 755 site/cache chmod -R 755 assets/avatars Probleem opgelost.

  • Gepost op: 2014-09-23
Kirby is een CMS gebaseerd op (markdown) bestanden, een zgn NoDB systeem. Dit betekent gemakkelijke beckup, idem aanmaken en wijzigen van postings via je standaard editor. Ik heb een implementatie gemaakt in ~/Sites/kirbycms en deze is oproepbaar via http://kirby.hahomail.nl. Slechts één wezenlijke modificatie was nodig om het systeem aan de praat te krijgen: ~/Sites/kirbycms/config/config.php // disable mod_rewrite on iMac/OS X c::set(&apos;rewrite&apos;, false); c::set(&apos;url&apos;, &apos;http://kirby.hahomail.nl&apos;); Dit was nodig omdat Apache (Mac OSX) problemen heeft met mod_rewrite. Stappen voor nieuwe iMac

  • Gepost op: 2013-10-02
Ik heb een nieuwe iMac besteld met een Fusion drive. Ik wil eigenlijk hierop een schone installatie doen. Maar een aantal applicaties wil ik gewoon overzetten, zoals Octopress en mijn pritlog/blog systeem. Aangezien het een half jaar geleden was dat ik voor het laatst iets met beide systemen deed, ben ik het overzicht over de diverse folders enigszins kwijt geraakt. Dat moet ik nu oppoetsen.
